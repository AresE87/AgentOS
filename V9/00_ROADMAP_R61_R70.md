# AgentOS v2 — Roadmap R61-R70: Enterprise profundo y nuevos paradigmas

**Fecha:** 29 de marzo de 2026
**Estado:** R1-R60 completas. v1.1 publicada con agent conversations, memory, personas, file understanding.
**Objetivo:** Features que hacen a AgentOS indispensable para empresas: el agente se integra con el stack de trabajo diario (email, calendario, bases de datos, APIs) y permite colaboración entre equipos humanos + agentes.

---

## Las 10 fases R61-R70

| Fase | Nombre | Qué agrega |
|------|--------|-----------|
| R61 | **Multi-user desktop** | Varios usuarios en la misma PC con agentes y datos separados |
| R62 | **Approval workflows** | El agente pide permiso antes de acciones críticas |
| R63 | **Calendar integration** | Google Calendar / Outlook: agendar, mover, cancelar reuniones |
| R64 | **Email integration** | Gmail / Outlook: leer, clasificar, responder, draft emails |
| R65 | **Database connector** | El agente consulta PostgreSQL, MySQL, MongoDB, SQLite directamente |
| R66 | **API orchestrator** | El agente llama APIs externas (REST/GraphQL) como herramientas |
| R67 | **Sandbox environments** | Ejecutar tareas riesgosas en un container Docker aislado |
| R68 | **Agent marketplace** | Comprar/vender agentes completos (no solo playbooks) |
| R69 | **Team collaboration** | Varios humanos comparten un pool de agentes con roles |
| R70 | **v1.2 — Enterprise release** | SSO avanzado (SCIM), provisioning, quotas por departamento |

**Regla:** Demo grabable. Cada feature resuelve un problema real de un equipo de trabajo.
