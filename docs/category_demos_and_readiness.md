# Category Demos And Readiness

## Demos canonicas del bloque C46-C50
- `billing-hardening`: usa eventos reales de upgrade y cambios de plan.
- `handoff-debugger`: usa handoffs y execution traces persistidos.
- `creator-economy`: usa creator projects, microtasks y escrows.
- `partner-distribution`: usa branding por org y catalogo aprobado para generar un bundle OEM.

## Comandos reales
- `cmd_category_demos`
- `cmd_definitive_readiness`
- `cmd_reliability_report`
- `cmd_partner_prepare_distribution`
- `cmd_creator_dashboard`

## Lectura honesta
- Si la base local no tiene eventos reales, los reportes quedan `partial` o `blocked`.
- El readiness definitivo no inventa cierre: refleja tablas, audit logs y bundles realmente presentes.
