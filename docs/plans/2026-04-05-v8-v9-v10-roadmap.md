## v8.0.0 — MARKETING AUTÓNOMO

**Lema:** "AgentOS se vende solo."

El usuario configura sus cuentas de redes sociales una vez. Un equipo de agentes genera contenido, publica, responde comentarios, monitorea menciones, y gestiona la presencia online — todo desde el Command Center. El usuario no toca redes sociales nunca más.

### Lo que se construye

#### M8-1: Social Media Connectors (Backend)
Crear `src-tauri/src/social/` con conectores reales:

- **Twitter/X** — OAuth 2.0, post tweets, reply, quote, like, search mentions
- **LinkedIn** — OAuth 2.0, publish posts, comment, share articles
- **Reddit** — OAuth 2.0, submit posts, comment in threads, monitor subreddits
- **Discord** (ya existe) — extender para publicar en servers externos
- **Hacker News** — HTTP scraping + comment via API
- **Product Hunt** — API para launch y respuestas
- **GitHub Discussions** — API para Q&A y anuncios
- **Instagram** — Meta Graph API (opcional, solo imágenes)

Cada connector implementa un trait `SocialPlatform`:
```rust
pub trait SocialPlatform: Send + Sync {
    fn name(&self) -> &str;
    async fn post(&self, content: &str, media: Option<&[u8]>) -> Result<PostResult>;
    async fn reply(&self, post_id: &str, content: &str) -> Result<PostResult>;
    async fn get_mentions(&self) -> Result<Vec<Mention>>;
    async fn get_engagement(&self) -> Result<EngagementMetrics>;
    async fn search(&self, query: &str) -> Result<Vec<SearchResult>>;
}
```

#### M8-2: Content Engine (Backend)
Crear `src-tauri/src/marketing/` con:

- **ContentGenerator** — genera posts, threads, artículos usando LLM
  - Adapta tono por plataforma (formal en LinkedIn, casual en Twitter, técnico en HN)
  - Genera imágenes con descripciones (el usuario puede conectar DALL-E o Midjourney API)
  - Calendario editorial: planifica contenido para toda la semana

- **EngagementManager** — monitorea y responde
  - Escanea menciones cada 15 minutos
  - Clasifica: pregunta, queja, elogio, spam
  - Genera respuesta apropiada
  - Si no está seguro → escala al usuario (handoff)

- **SEOAnalyzer** — optimiza contenido
  - Analiza keywords relevantes
  - Sugiere títulos y hashtags
  - Monitorea posicionamiento

- **CampaignManager** — coordina campañas multi-plataforma
  - "Lanzar campaña: publicar en Twitter, LinkedIn, Reddit, y HN"
  - Timing inteligente (publica a la hora de más engagement por plataforma)
  - A/B testing de títulos
  - Métricas agregadas

#### M8-3: Marketing Command Center (Frontend)
Nueva página `Marketing` en el sidebar con:

- **Dashboard de redes** — followers, engagement, mentions por plataforma
- **Calendario editorial** — vista semanal/mensual de contenido planificado
- **Generador de contenido** — escribe prompt → el agente genera variantes para cada red
- **Bandeja de menciones** — timeline unificado de todas las menciones, con respuestas sugeridas
- **Campañas** — crear, ejecutar, medir campañas multi-plataforma
- **Analytics** — engagement rate, best performing posts, growth trends

#### M8-4: Marketing Agents (Coordinator Integration)
5 specialist agents nuevos que trabajan en el Command Center:

1. **Social Media Manager** — decide qué publicar, cuándo, y dónde
2. **Content Writer** — genera textos adaptados por plataforma
3. **Community Manager** — responde menciones y comentarios
4. **SEO Specialist** — optimiza contenido para búsqueda
5. **Analytics Analyst** — reporta métricas y sugiere mejoras

Template de misión: "Lanzamiento de producto"
→ El Social Media Manager planifica → Content Writer genera 10 posts → se publican en 5 plataformas → Community Manager responde feedback → Analytics reporta resultados.

#### M8-5: Self-Promotion Mode
AgentOS se promueve a sí mismo:

- Al instalar, pregunta: "¿Querés que AgentOS gestione su propia presencia en redes?"
- Si dice sí: crea cuentas de redes para el producto del usuario (o usa las del usuario)
- Publica sobre las capacidades de AgentOS
- Responde preguntas en foros técnicos
- Genera threads educativos sobre automatización
- Monitorea competidores y posiciona

**IPC Commands (~20):**
- Social CRUD: connect_platform, disconnect, list_platforms, post, reply, get_mentions
- Content: generate_content, schedule_post, get_calendar, create_campaign
- Analytics: get_social_analytics, get_engagement, get_growth

**Sesiones de implementación: 10**
- 4 backend (connectors, content engine, engagement, campaigns)
- 4 frontend (marketing page, calendar, mentions, analytics)
- 2 polish (agents, self-promotion, templates)

## v9.0.0 — CREATOR ECONOMY

**Lema:** "Tu conocimiento operativo tiene precio."

El usuario que sabe hacer algo específico (procesar facturas, gestionar CRM, automatizar reportes) puede empaquetar ese conocimiento como un "Training" — un workflow entrenado con ejemplos reales que otros usuarios pueden comprar e instalar.

### La visión económica

```
CREADOR                              COMPRADOR
┌─────────────┐                     ┌─────────────┐
│ Sabe hacer   │    $4.99/mes        │ Necesita     │
│ facturas     │───────────────────▶ │ automatizar  │
│              │    Training Pack     │ facturas     │
│ Lo empaqueta │◀───────────────────│              │
│ como Training│    Rating ★★★★★     │ Lo instala   │
└─────────────┘                     └─────────────┘
        │                                   │
        │ 70% revenue                       │ Funciona
        │ ($3.49)                            │ al instante
        ▼                                   ▼
   Creator Dashboard                 Su AgentOS ejecuta
   con métricas                      la tarea perfectamente
```

### Lo que se construye

#### E9-1: Training Recorder (Backend)
Crear `src-tauri/src/training_studio/` con:

- **TrainingRecorder** — graba la ejecución de una tarea como "training"
  - El usuario hace la tarea manualmente (o guiado por el agente)
  - El sistema captura: prompts, tool calls, decisiones, correcciones
  - Genera un "Training Pack" que otro agente puede reproducir

- **TrainingPack** struct:
  ```rust
  pub struct TrainingPack {
      pub id: String,
      pub title: String,
      pub description: String,
      pub category: String,
      pub creator_id: String,
      pub version: String,
      pub examples: Vec<TrainingExample>,    // input→output pairs
      pub workflow: SmartPlaybook,            // el playbook generado
      pub system_prompt_additions: String,    // conocimiento especializado
      pub tools_required: Vec<String>,
      pub tags: Vec<String>,
      pub price_monthly: Option<f64>,        // None = gratis
      pub price_one_time: Option<f64>,
      pub created_at: String,
  }

  pub struct TrainingExample {
      pub input: String,
      pub expected_output: String,
      pub tool_calls: Vec<ToolCallRecord>,
      pub corrections: Vec<String>,          // correcciones del usuario
  }
  ```

- **TrainingPlayer** — ejecuta un training pack
  - Usa los examples como few-shot learning
  - Aplica el system_prompt_additions
  - Ejecuta el workflow con las tools requeridas
  - Si algo falla → aplica correcciones del training

#### E9-2: Creator Studio (Frontend)
Nueva página `Studio` en el sidebar:

- **Recorder** — botón "Grabar Training" → el usuario hace la tarea → el sistema captura todo
- **Editor** — editar los examples, agregar correcciones, ajustar el workflow
- **Preview** — probar el training antes de publicar
- **Pricing** — configurar precio (gratis, mensual, o pago único)
- **Publisher** — publicar al marketplace con screenshots, descripción, demo

#### E9-3: Marketplace 2.0 (Backend + Frontend)
Evolucionar el marketplace existente:

- **Categorías:** Finanzas, Legal, Marketing, Desarrollo, Operaciones, Datos, Educación, Custom
- **Buscar por problema:** "automatizar facturas" → muestra trainings relevantes
- **Ratings y reviews** reales (ya existe SQLite, mejorarlo)
- **Preview antes de comprar** — ver los steps del training sin comprarlo
- **Versioning** — los creadores pueden actualizar sus trainings
- **Revenue sharing** — 70% creator / 30% plataforma (configurable)

#### E9-4: Payments + Creator Dashboard
- **Stripe Connect** para pagos a creadores (ya hay Stripe base de v5)
- **Creator Dashboard:**
  - Earnings: MRR, total revenue, trending trainings
  - Downloads: por training, por día
  - Reviews: promedio, recientes
  - Subscribers: lista de compradores activos
  - Payouts: historial de pagos recibidos

#### E9-5: Training Quality System
- **Auto-validation** — el sistema ejecuta el training 3 veces con inputs de prueba antes de permitir publicar
- **Buyer protection** — si el training falla >30% de las veces, se pausa y se notifica al creador
- **Versioning obligatorio** — cada update del training incrementa versión
- **Ratings ponderados** — más peso a ratings de compradores verificados

**IPC Commands (~25):**
- Studio: start_recording, stop_recording, save_training, edit_training, preview_training
- Marketplace: list_trainings, search_trainings, buy_training, install_training, rate_training
- Creator: get_creator_dashboard, get_earnings, get_downloads, request_payout
- Player: run_training, get_training_status

**Sesiones: 10**
- 3 backend (recorder, player, marketplace 2.0)
- 3 frontend (studio, marketplace, creator dashboard)
- 2 payments (Stripe Connect, payouts)
- 2 polish (quality system, versioning, templates)

## v10.0.0 — PRODUCTION READY

**Lema:** "Listo para el mundo."

v10 no agrega features. Pule TODO lo existente hasta que sea bulletproof. Después de v10, el usuario (vos) solo se dedica a narrativa y tracción — el producto se sostiene solo.

### Lo que se hace

#### P10-1: Stability Hardening
- Crash recovery: si la app crashea, se reinicia automáticamente sin perder estado
- Session recovery: las misiones en progreso se reanudan después de reiniciar
- Container auto-restart: si un worker Docker muere, se recrea
- DB corruption handling: WAL checkpoints automáticos, backup diario
- Memory leak detection: monitoreo de heap, alert si >500MB

#### P10-2: Performance Optimization
- Startup time < 3 segundos (lazy loading de módulos pesados)
- Chat response perception < 500ms (streaming + skeleton UI)
- Docker container startup < 5 segundos (image pre-warmed)
- Frontend bundle < 500KB (tree-shaking, lazy routes)
- SQLite query optimization (EXPLAIN ANALYZE en queries lentas)

#### P10-3: Security Final Audit
- Penetration testing: probar TODOS los endpoints de la API
- API key rotation: poder rotar keys sin downtime
- Container escape prevention: verificar que Docker config es segura
- Secrets scanning: CI que detecta API keys en el código
- CSP headers en la API
- Rate limiting por IP (no solo por API key)

#### P10-4: Documentation & Onboarding
- Video tutoriales embebidos en la app (5 videos de 60 segundos)
- Tooltips contextuales en cada sección
- "Tour guiado" interactivo al abrir cada página por primera vez
- FAQ dinámico que se actualiza basado en preguntas frecuentes
- Centro de ayuda con búsqueda

#### P10-5: Monitoring & Analytics
- Dashboard de salud del producto (no del usuario — del producto mismo)
- Tracking de adoption metrics (opt-in)
- Crash reports anónimos
- Feature usage analytics (qué features se usan más)
- Feedback loop: NPS survey a los 7, 30, 90 días

#### P10-6: Distribution
- Windows installer firmado (code signing certificate)
- Auto-update robusto (delta updates, rollback)
- Landing page v3 con video demo
- Product Hunt launch ready
- Press kit: screenshots, logo, copy, one-pager

#### P10-7: Marketing Bootstrap
- Los agentes de v8 ya están corriendo
- Configurar las cuentas de redes para el lanzamiento
- Pre-generar 30 días de contenido
- Monitorear 5 subreddits y 3 foros relevantes
- Landing page con waitlist antes del launch

**Sesiones: 10**
- 3 stability (crash recovery, session recovery, container restart)
- 2 performance (startup, bundle, Docker warmup)
- 2 security (audit, rotation, CSP)
- 1 documentation (videos, tooltips, tour)
- 1 distribution (signing, auto-update, landing)
- 1 marketing bootstrap (agents configured, content pre-generated)

## v11.0.0 — AGENT TEAMS AS A SERVICE

**Lema:** "Tu equipo de marketing/ventas/soporte, listo en 5 minutos."

v8 construye el equipo de marketing para AgentOS. v11 lo convierte en un PRODUCTO: cualquier empresa puede abrir AgentOS, elegir "Equipo de Marketing", configurar sus cuentas de redes, y tener un equipo de agentes trabajando para ellos.

### Teams pre-configurados (Templates de equipos)

Cada team es un conjunto de agentes + connectors + workflows listos para usar:

#### 1. Marketing Team
- Social Media Manager (programa y publica)
- Content Writer (genera posts, blogs, threads)
- Community Manager (responde menciones)
- SEO Specialist (optimiza contenido)
- Analytics Analyst (reporta métricas)
- **Connectors:** Twitter, LinkedIn, Reddit, Instagram, HN
- **Output:** Calendario editorial, posts publicados, métricas de engagement

#### 2. Sales Team
- Lead Researcher (busca prospects en LinkedIn, directorios)
- Proposal Writer (genera propuestas personalizadas)
- Email Outreach Agent (envía secuencias de cold email)
- CRM Manager (actualiza pipeline)
- Follow-up Agent (agenda calls, envía reminders)
- **Connectors:** LinkedIn, Gmail, Calendar, CRM API
- **Output:** Pipeline de ventas, propuestas, meetings agendados

#### 3. Support Team
- Ticket Triage Agent (clasifica y prioriza)
- L1 Support Agent (responde preguntas frecuentes)
- Documentation Writer (genera artículos de help center)
- Escalation Manager (escala a humano cuando necesita)
- Satisfaction Monitor (sigue NPS y feedback)
- **Connectors:** Email, Discord/Slack, Knowledge base
- **Output:** Tickets resueltos, documentación, CSAT score

#### 4. Content Team
- Research Agent (investiga temas)
- Writer (redacta artículos, whitepapers)
- Editor (revisa y mejora)
- SEO Optimizer (keywords, meta tags)
- Distribution Agent (publica en blog, Medium, newsletter)
- **Connectors:** WordPress API, Medium API, Mailchimp/Resend
- **Output:** Artículos publicados, newsletter enviado

#### 5. Finance Team
- Invoice Processor (lee y registra facturas)
- Expense Categorizer (clasifica gastos)
- Report Generator (P&L mensual, cashflow)
- Tax Preparer (organiza para declaración)
- Payment Reminder (envía recordatorios de cobro)
- **Connectors:** Gmail (facturas), Calendar (vencimientos), Excel/Sheets
- **Output:** Reportes financieros, facturas procesadas

### Implementación

#### T11-2: Team Setup Wizard (Frontend)
Flujo de 3 pasos:
1. Elegir team → ver qué agentes incluye
2. Conectar cuentas → OAuth para cada plataforma requerida
3. Configurar → tono de voz, frecuencia de publicación, horarios
→ Team listo y corriendo en 5 minutos

#### T11-3: Team Dashboard (Frontend)
Cada team tiene su propia vista con:
- Agentes activos y su estado
- Métricas del team (posts publicados, leads generados, tickets resueltos)
- Timeline de acciones
- Configuración del team

#### T11-4: Autonomous Scheduling
Los teams corren en background sin intervención:
- Marketing: publica 3 veces por semana automáticamente
- Sales: busca leads y envía outreach diariamente
- Support: monitorea inbox y responde 24/7
- El usuario solo ve los resultados y aprueba lo importante

#### T11-5: Team Marketplace
Los creadores de v9 pueden crear y vender Teams personalizados:
- "Equipo de Marketing para Restaurantes"
- "Equipo de Ventas para SaaS B2B"
- "Equipo de Soporte para E-commerce"

## v12.0.0 — AUTONOMOUS BUSINESS OPERATING SYSTEM

**Lema:** "Tu empresa corre en piloto automático."

v12 integra TODO: los teams de v11, la economy de v9, el marketing de v8, los workers Docker de v7, el coordinator de v6 — en un sistema operativo de negocio completo.

### Lo que agrega v12

#### B12-1: Business Dashboard
Una vista ejecutiva que muestra TODO el negocio:
- Marketing: followers, engagement, leads generados (del Marketing Team)
- Ventas: pipeline, revenue, conversion rate (del Sales Team)
- Finanzas: cashflow, gastos, P&L (del Finance Team)
- Soporte: tickets abiertos, CSAT, tiempo de respuesta (del Support Team)
- Marketplace: trainings vendidos, revenue de creador, ratings
- Todo en UNA pantalla con gráficos y KPIs en tiempo real

#### B12-2: Inter-Team Orchestration
Los teams hablan entre sí:
- Marketing genera un lead → Sales lo toma automáticamente
- Sales cierra un deal → Finance genera la factura
- Un cliente tiene un problema → Support lo escala al team correcto
- Finance detecta un pago pendiente → Sales envía follow-up

```
Marketing Team ──(lead)──▶ Sales Team ──(deal)──▶ Finance Team
                                │                       │
                            (issue)                  (invoice)
                                ▼                       ▼
                          Support Team            Customer notified
```

#### B12-3: Business Automations (Triggers)
Reglas de negocio que el usuario define en lenguaje natural:
- "Si un lead no responde en 3 días, enviar follow-up automático"
- "Si el cashflow baja de $5000, alertar y pausar gastos no esenciales"
- "Si un post tiene más de 100 likes, crear un hilo expandiendo el tema"
- "Si llega una factura por email, procesarla y categorizarla automáticamente"

#### B12-4: Revenue Analytics
Para creadores del marketplace:
- Dashboard unificado de TODOS los revenue streams:
  - Trainings vendidos
  - Teams vendidos
  - Subscripciones activas
  - Payouts pendientes
- Proyecciones de revenue
- Top performers (qué trainings venden más)
- Competitor analysis (qué trainings similares existen)

#### B12-5: White-Label Business OS
Las empresas pueden brandear AgentOS como su propio producto:
- Cambiar nombre, logo, colores
- Elegir qué teams incluir
- Configurar pricing propio
- Distribuir a sus clientes como SaaS propio

**Sesiones: 10**
