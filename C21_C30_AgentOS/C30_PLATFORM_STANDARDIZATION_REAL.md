# C30 — PLATFORM STANDARDIZATION REAL

## Objetivo
Consolidar AgentOS como plataforma coherente a nivel técnico: comandos, contratos, estados y naming.

## Qué implementar
1. Auditoría de comandos IPC/API expuestos.
2. Estados y enums consistentes.
3. Contratos/tipos normalizados.
4. Documento de estándar interno:
   - naming
   - estados
   - errores
   - versionado

## Definition of Done
- existe un estándar técnico interno usable
- los módulos nuevos y existentes más críticos lo siguen
- baja fricción de mantenimiento

## Demo obligatoria
“Documento + 3 áreas normalizadas de verdad”.

## No hacer
- proclamar ‘estándar global’ sin primero ordenar el estándar interno
